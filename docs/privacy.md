# Datenschutz-Grundprinzipien

> **Diese Prinzipien gelten fuer ALLE zukuenftigen Funktionen — keine Ausnahmen.**
> Bei jedem neuen Feature zuerst pruefen: verletzt es eines dieser Prinzipien?

- **DSGVO-konform by Design** — kein Nachbessern, Datenschutz ist von Anfang an eingebaut
- **Server speichert NICHTS** ausser: dem Arego-ID-Hash fuer Routing (kein Klartext, kein Profil)
- **Keine Metadaten** — kein "wer mit wem", kein "wann", kein "wie viel"
- **Keine IP-Adressen** speichern oder loggen
- **Alle Nachrichten, Fotos, Dateien** werden ausschliesslich P2P direkt zwischen Geraeten uebertragen
- **Kein Tracking, keine Analysen, keine Werbung**
- **Nutzer loescht App = alles weg** — nichts bleibt auf dem Server
- **E2E-Verschluesselung ueberall** — der Server sieht niemals Inhalte
- **Datensparsamkeit** — nur das absolute Minimum speichern
- **Jede neue Funktion** muss diese Prinzipien einhalten — keine Ausnahmen
- **Kostendeckend, nicht gewinnorientiert** — kein Upselling, keine Premium-Features die Datenschutz einschraenken
